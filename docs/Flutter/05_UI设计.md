# 05_UI 设计

## 1. 设计哲学

Flutter 版本的 UI 设计必须与当前 LifeOS PWA 品牌一致，同时充分利用原生移动端的体验优势。

核心原则：

- 极简：第一屏只解决一个问题。
- 安静：UI 不争夺注意力，AI 只在必要时出现。
- 温暖：关系产品需要情感化，不是冰冷的效率工具。
- 移动优先：所有设计从单手操作出发。
- Decision Dashboard：不是信息展示，而是行动入口。

## 2. Design System

### 2.1 Color

延续 LifeOS 品牌色调，支持 Light / Dark Mode。

主要颜色：

- background: 白色 / 深色 #0F0F0F
- surface: 浅灰 / 深灰 #1C1C1E
- primary: 柔和紫色 #7C5CFF
- primaryContainer: 浅紫容器
- textPrimary: 主文字
- textSecondary: 次要文字
- textTertiary: 第三层文字
- border: 边框
- success: 绿色
- warning: 琥珀色
- danger: 红色

设计原则：

- 主色使用柔和紫色，传递温暖、智能、可信。
- 避免高饱和色，减少视觉压力。
- Dark Mode 不是简单反色，而是重新调整对比度。

### 2.2 Typography

使用系统字体优先：

- iOS: SF Pro
- Android: Roboto

字号层级：

- display: 32px bold, 用于首页大标题
- headline: 24px bold, 页面标题
- titleLarge: 20px semibold, Card 标题
- titleMedium: 17px semibold, 列表标题
- bodyLarge: 17px regular, 正文
- bodyMedium: 15px regular, 默认正文
- bodySmall: 13px regular, 辅助说明
- caption: 12px medium, 标签、时间
- button: 16px semibold, 按钮文字

### 2.3 Spacing

基于 4px 网格：

- xs: 4px
- sm: 8px
- md: 16px
- lg: 24px
- xl: 32px
- xxl: 48px

设计原则：

- Card 内边距统一 16px。
- 页面水平边距 16px（移动端）。
- 元素之间最小间距 8px。

## 3. Card

Card 是 Flutter 版本最重要的 UI 元素。

Today One Thing Card 设计：

- 圆角 20px
- 柔和阴影（Elevation 2-4）
- 内边距 20px
- 顶部显示人物头像与名字
- 中间显示原因与建议行动
- 底部三个 Action Button

Card 原则：

- 每个 Card 只传递一个决策。
- 不堆叠多个 Card。
- 使用留白而不是边框分隔。

## 4. Bottom Navigation

首页底部导航最多 3-4 个入口：

- Home：Today One Thing
- People：人物列表
- Capture：快速记忆捕获
- Settings：设置

设计原则：

- 图标简洁、含义明确。
- 当前选中项使用 primary 色。
- 不显示红点/badge，避免制造焦虑。

## 5. Motion

动画必须 subtle、自然、不打扰。

允许的动画：

- 页面切换：滑动过渡 200-300ms
- Card 出现：淡入 + 轻微上移 200ms
- 按钮按下：缩放 0.96
- 列表加载：骨架屏，不要旋转 loading

禁止的动画：

- 复杂庆祝动画
- 弹跳、旋转、闪烁
- 长时间过渡

Motion 原则：动画应该帮助用户理解状态变化，而不是吸引注意力。

## 6. Haptic

适度使用 Haptic Feedback：

- 完成 Today One Thing 时：轻微成功反馈
- 长按 Capture 时：轻微触发反馈
- 滑动切换人物时：轻微 tick

避免：

- 每次点击都震动
- 强烈或持续震动
- 与情感不匹配的重反馈

## 7. Dark Mode

Dark Mode 不是简单反色，而是重新设计：

- 背景使用纯黑或接近纯黑，不是深灰
- 文字使用不同层级的白色/灰色
- Card 使用略亮的表面色，与背景区分
- 主色在 Dark Mode 下更亮，保持可访问性
- 图片/头像添加轻微暗角，避免刺眼

## 8. 品牌一致性

Flutter 版本的 UI 必须与 PWA 保持：

- 相同的颜色系统
- 相同的字体层级
- 相同的圆角与间距
- 相同的交互语言
- 相同的情感调性

差异点：

- Flutter 可以使用更原生的手势与动画
- Flutter 可以更好地利用底部 Sheet、Haptic
- Flutter 的首页可以更沉浸

## 9. 总结

Flutter 的 UI 设计不是重新发明视觉语言，而是把 PWA 已经验证的设计系统，在原生移动端做到极致。目标是让用户感到熟悉、温暖、安静，每一次打开都只有一种感觉：我知道今天该做什么。
