# 06_推送方案

## 1. 核心原则

LifeOS 的推送不是营销工具，而是关系守护的延伸。推送必须满足：

- 有价值：每一条推送都必须让用户觉得幸好收到这条。
- 可解释：用户必须知道为什么现在收到这条推送。
- 可控制：用户可以随时关闭任何类型的推送。
- 不打扰：默认安静，只在真正必要时出现。

## 2. 推送类型

### 2.1 Today One Thing 推送

触发条件：

- Why Now Engine 判断今天有一个人需要用户关注。
- 用户设置的最佳时间窗口（默认早上 9:00，可调整）。

推送内容示例：

- 标题：今天有一个人需要你
- 正文：Alice - 45 天没有联系，她最近提到工作压力很大
- 按钮：查看 / 已联系 / 稍后

频率：

- 每天最多 1 次。
- 如果没有高优先级关系，不推送。

### 2.2 Birthday Reminder

触发条件：

- 重要关系人物生日前 1-3 天。
- 根据关系亲密度调整提前量。

推送内容示例：

- 标题：Alice 的生日还有 2 天
- 正文：你之前记过她喜欢百合，要不要提前准备一句祝福？
- 按钮：写祝福 / 忽略

### 2.3 Gentle Reminder

触发条件：

- 用户承诺了某件事，临近 deadline。
- 或者某段重要关系长期未维护，且 Today One Thing 未被处理。

推送内容示例：

- 标题：你答应 Bob 这周末发方案
- 正文：已经过去 3 天了
- 按钮：跟进 / 稍后

原则：

- 不审判用户。
- 语气温和。
- 提供一键行动。

### 2.4 Local Notification vs Remote Push

Local Notification：

- 由本地计算生成。
- 典型场景：Today One Thing、Birthday、Follow-up。

Remote Push：

- 由 Backend 推送。
- 典型场景：跨设备同步后的新提醒、重要 AI 发现。

设计原则：

- 绝大多数推送由本地生成，不依赖网络。
- Remote Push 仅用于跨设备通知和 Backend 端发现的关键事件。

## 3. iOS Push

技术方案：

- 使用 Apple Push Notification Service (APNs)。
- Backend 使用 APNs Provider API 发送远程推送。
- Flutter 使用 firebase_messaging 或 flutter_local_notifications 处理。

权限策略：

- 首次安装不立即请求推送权限。
- 在用户体验到 Today One Thing 价值后，再引导开启推送。
- 如果用户拒绝，不反复请求。

推送分组：

- TODAY_ONE_THING
- BIRTHDAY_REMINDER
- GENTLE_REMINDER

每个 Category 对应不同的 Action Buttons。

## 4. Android Push

技术方案：

- 使用 Firebase Cloud Messaging (FCM)。
- 高优先级消息用于 Today One Thing。
- 普通优先级用于生日提醒等可延迟通知。

权限策略：

- Android 13 及以上需要显式请求通知权限。
- 同样采用延迟请求策略。

渠道管理：

创建独立 Notification Channel：

- today_one_thing：高优先级
- birthday_reminder：默认优先级
- gentle_reminder：默认优先级
- sync_updates：低优先级

## 5. Today One Thing 推送细节

最佳推送时间：

- 默认早上 9:00。
- 用户可自定义为早上、中午或傍晚。
- 根据用户历史响应时间学习最佳时机。

通知 Action：

- 已联系：标记完成，可触发简短记录。
- 稍后：1-3 小时后再次提醒。
- 忽略：AI 学习这个提醒不重要。

通知展开：

- 展开后显示建议行动的具体文案。
- 显示 Why Now 简短解释。

## 6. Birthday Reminder 推送细节

提前量策略：

- 家人/伴侣：提前 3-7 天
- 亲密朋友：提前 2-3 天
- 普通朋友/同事：提前 1 天

内容策略：

- 结合用户过往记录的偏好。
- 提供祝福语草稿。
- 不直接推荐购买礼物。

## 7. Gentle Reminder 推送细节

触发阈值：

- 承诺 deadline 前 24 小时。
- Today One Thing 被忽略超过 48 小时。
- 某段重要关系超过 60 天无 meaningful interaction。

频率控制：

- 同一个人每周最多 1 次 Gentle Reminder。
- 避免连续多天推送。

语气：

- 温和，不责备。
- 提供帮助，而不是施加压力。

## 8. 推送频率上限

| 类型 | 最大频率 |
|------|---------|
| Today One Thing | 每天 1 次 |
| Birthday Reminder | 每人每年 1 次 |
| Gentle Reminder | 每周最多 2-3 次 |
| Remote Push | 每月最多 1-2 次 |

总原则：用户平均每天收到的 LifeOS 推送不超过 1 条。

## 9. 用户控制

设置中提供细粒度控制：

- 总通知开关
- Today One Thing 开关
- Birthday Reminder 开关
- Gentle Reminder 开关
- Weekly Brief 开关（默认关闭）
- 推送时间窗口

## 10. 总结

Flutter 的推送方案遵循 LifeOS 的核心原则：安静、有价值、可解释、可控制。推送不是产品的存在理由，而是关系守护在移动端的自然延伸。

> 最好的推送，是用户收到后感到温暖而不是被打扰的推送。
