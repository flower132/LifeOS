# 08_开发路线图

## 1. 总体原则

Flutter 开发不是当前重点，只有当 PWA 验证成功后才会正式启动。本路线图用于指导未来 Flutter 项目启动后的开发节奏。

**核心原则：**

- 每一阶段都必须有可验证的完成标准。
- 优先复用 PWA 已验证的 Backend、Database、AI Prompt。
- Flutter 前端只负责展示与交互，不重复实现业务逻辑。
- 每阶段结束后评估是否继续投入。

## 2. 阶段划分

### 阶段一：项目初始化（未来）

**目标：** 搭建 Flutter 项目基础架构，确保团队可以快速开发。

**任务：**
- 初始化 Flutter 项目（iOS + Android）。
- 配置环境：Dev / Staging / Prod。
- 集成必要依赖：Riverpod、GoRouter、Drift、flutter_local_notifications、firebase_messaging、http、dio。
- 配置 CI/CD：GitHub Actions / Codemagic。
- 搭建目录结构：screens / services / repositories / models / widgets / utils。
- 配置 Theme、Color、Typography。

**完成标准：**
- 项目能在 iOS 模拟器和 Android 模拟器上运行。
- 空首页可以正常显示。
- CI/CD 能自动构建。

**依赖 PWA：** 无，但需与 PWA 团队对齐 Design System。

---

### 阶段二：认证与账户（未来）

**目标：** 实现用户登录与数据同步基础。

**任务：**
- 集成 Supabase Auth（与 PWA 一致）。
- 实现登录 / 注册 / 登出。
- 实现 Token 安全存储（Flutter Secure Storage）。
- 实现匿名登录或 Guest Mode（可选）。
- 用户首次登录后拉取云端数据。

**完成标准：**
- 用户可以登录并看到同步的数据。
- Token 安全存储，不会丢失。
- 登出后本地数据清除。

**依赖 PWA：** Auth API、User Schema。

---

### 阶段三：首页 Today One Thing（未来）

**目标：** 实现 Decision Dashboard 首页，展示 Today One Thing。

**任务：**
- 实现 HomeScreen。
- 实现 TodayOneThingCard 组件。
- 集成 Backend API `/ai/today-one-thing`。
- 实现三个 Action Button：已联系 / 稍后 / 忽略。
- 实现"为什么现在？"展开页。
- 实现无提醒时的空状态 UI。
- 集成本地通知调度。

**完成标准：**
- 打开 App 3 秒内显示 Today One Thing。
- 用户可以与 Today One Thing 交互。
- 无提醒时显示正确空状态。

**依赖 PWA：** Today One Thing API、Why Now Engine、Reminder Schema。

---

### 阶段四：Memory Capture（未来）

**目标：** 实现低摩擦的记忆捕获体验。

**任务：**
- 实现底部 Capture Sheet。
- 实现语音输入（speech_to_text）。
- 实现文字输入。
- 实现聊天记录分享入口。
- 调用 Backend API `/ai/analyze-memory`。
- 本地保存 Memory 到 SQLite。
- 实现 Memory 列表展示。

**完成标准：**
- 用户可以用语音或文字在 30 秒内记录一条 Memory。
- Memory 保存在本地并同步到云端。
- AI 分析后可能生成 Reminder。

**依赖 PWA：** Memory API、Analyze Memory API、Memory Schema。

---

### 阶段五：Relationship Guardian（未来）

**目标：** 实现人物关系管理与长期记忆上下文。

**任务：**
- 实现 Person List Screen。
- 实现 Person Detail Screen（记忆流 + AI Chat 入口）。
- 实现 Timeline 自动生成展示。
- 实现人物创建与编辑。
- 实现重要性标记。

**完成标准：**
- 用户可以查看所有人物。
- 可以查看某个人物的记忆时间线。
- 可以从人物详情进入 AI Chat。

**依赖 PWA：** Person API、Memory API、Timeline API。

---

### 阶段六：AI Chat（未来）

**目标：** 实现基于长期关系上下文的 AI Chat。

**任务：**
- 实现 AIChatScreen。
- 实现消息气泡 UI。
- 集成 Backend API `/ai/chat`。
- 实现聊天历史本地缓存。
- 支持基于某个人物的上下文聊天。
- 实现 AI 承认自己不知道的情况。

**完成标准：**
- 用户可以与 AI 对话。
- AI 能基于长期记忆回答关于某人的问题。
- 聊天记录可持久化。

**依赖 PWA：** AI Chat API、AI Conversation Schema。

---

### 阶段七：Notification（未来）

**目标：** 实现完整的推送与本地通知系统。

**任务：**
- 集成 firebase_messaging（Remote Push）。
- 集成 flutter_local_notifications（Local Notification）。
- 实现 Today One Thing 推送。
- 实现 Birthday Reminder 推送。
- 实现 Gentle Reminder 推送。
- 实现通知权限管理。
- 实现通知 Action Buttons。

**完成标准：**
- iOS / Android 都能正常接收推送。
- Today One Thing 可以本地调度提醒。
- 用户可以通过通知 Action 快速响应。

**依赖 PWA：** Backend Push 能力、Reminder Schema。

---

### 阶段八：上线（未来）

**目标：** 完成 Flutter App 的正式发布。

**任务：**
- 完成 App Store / Google Play 资料。
- 完成隐私政策 / 服务条款。
- 完成测试（内部测试、封闭测试）。
- 修复所有 P0/P1 Bug。
- 提交 App Store / Google Play 审核。
- 上线后监控崩溃、留存、用户反馈。

**完成标准：**
- App 在 App Store 和 Google Play 上线。
- 7 日留存率达到目标。
- 崩溃率 < 1%。

**依赖 PWA：** 产品方向已验证、Backend 稳定、AI Prompt 定型。

## 3. 路线图依赖关系

```
阶段一：项目初始化
    ↓
阶段二：认证与账户
    ↓
阶段三：首页 Today One Thing
    ↓
阶段四：Memory Capture
    ↓
阶段五：Relationship Guardian
    ↓
阶段六：AI Chat
    ↓
阶段七：Notification
    ↓
阶段八：上线
```

**关键依赖：** 阶段三之后的所有阶段都依赖 PWA 提供的稳定 Backend API 和 Why Now Engine。

## 4. 启动 Flutter 的硬性条件

以下五个条件必须全部满足，才能从阶段一开始：

1. **PWA MVP 完成**
   - Today One Thing
   - Why Now Engine
   - One-Tap Action
   - Memory Capture
   - Relationship Memory
   - AI Chat

2. **已有真实用户验证产品价值**
   - 7 日留存率 > 30%
   - Today One Thing 执行率 > 40%
   - 用户主动记录 Memory 频率 > 每周 2 次

3. **核心数据模型稳定**
   - Person / Memory / Reminder / Timeline / AI Conversation / Settings Schema 不再大幅变动
   - 数据库 Schema 已跑通至少一个真实使用周期

4. **AI Prompt 基本定型**
   - Why Now Engine 的 Prompt 策略稳定
   - AI Chat 的上下文管理方案稳定
   - 模型调用成本可控

5. **产品进入正式移动端发布阶段**
   - 团队决定需要原生 App 形态
   - 有足够开发资源维护 Flutter
   - 准备承担 App Store / Google Play 的发布与运营成本

## 5. 风险控制

| 风险 | 缓解措施 |
|------|---------|
| PWA 验证失败 | 不启动 Flutter |
| Backend API 不稳定 | 延迟 Flutter 开发，优先稳定 Backend |
| 开发资源不足 | 先完成阶段一~三，验证最小可行原生体验 |
| App Store 审核被拒 | 提前准备隐私政策与内容安全策略 |
| 用户不买账原生 App | 通过 TestFlight 小规模验证后再全面上线 |

## 6. 总结

Flutter 的路线图不是独立计划，而是 PWA 成功后的放大器。只有当 PWA 证明了 LifeOS 对用户有价值时，Flutter 才有意义。

> **PWA 验证价值，Flutter 放大体验。在价值未被验证之前，Flutter 只存在于文档中。**
