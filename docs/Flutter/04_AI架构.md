# 04_AI 架构

## 1. 核心原则

Flutter 版本**不直接调用任何 LLM**。所有 AI 能力统一通过 LifeOS Backend API 获取。

设计原则：

- **模型无关**：Backend 可以自由切换底层模型，Flutter 无需改动。
- **成本可控**：Backend 统一管理模型调用、缓存、降级、限流。
- **隐私安全**：敏感关系数据不直接流向第三方 LLM，由 Backend 统一处理与脱敏。
- **体验一致**：PWA 与 Flutter 共享同一套 AI 能力。
- **可回退**：Backend 不可用时，Flutter 有本地默认行为（如显示已缓存的 Reminder）。

## 2. 架构分层

```
Flutter App
    │
    │ HTTPS / JSON
    ▼
LifeOS Backend API
    │
    ├──────────────┬──────────────┬──────────────┐
    ▼              ▼              ▼              ▼
  OpenAI        Claude         Gemini         预留模型
  (Primary)    (Fallback)    (Vision)       DeepSeek / Qwen
```

## 3. Flutter AI Layer

Flutter 中的 AI Layer 非常轻薄，只负责：

- 构造请求参数。
- 调用 Backend API。
- 解析结构化响应。
- 缓存 AI 输出。
- 处理错误与降级。

### 3.1 AIService 接口

```dart
class AIService {
  final BackendClient _client;

  AIService(this._client);

  /// 获取今天的 Today One Thing
  Future<TodayOneThing?> getTodayOneThing();

  /// 获取某个关系的 Why Now 解释
  Future<WhyNowExplanation> explainWhyNow(String personId);

  /// AI Chat：基于长期关系上下文回答问题
  Future<AIChatMessage> chat({
    required String? personId,
    required String message,
    List<AIChatMessage>? history,
  });

  /// 提交新 Memory 后，请求 AI 分析并生成/更新 Reminder
  Future<List<ReminderSuggestion>> analyzeMemory(Memory memory);

  /// 获取 AI 对某个人关系的长期总结（用于 Chat 上下文）
  Future<RelationshipSummary> getRelationshipSummary(String personId);

  /// 获取 Weekly Brief（可选，默认关闭）
  Future<WeeklyBrief?> getWeeklyBrief();
}
```

### 3.2 Backend API 端点

| 端点 | 方法 | 用途 |
|------|------|------|
| `/ai/today-one-thing` | GET | 获取今日推荐 |
| `/ai/why-now/:personId` | GET | 解释为什么现在 |
| `/ai/chat` | POST | AI Chat |
| `/ai/analyze-memory` | POST | 分析新记忆 |
| `/ai/relationship-summary/:personId` | GET | 关系摘要 |
| `/ai/weekly-brief` | GET | 每周简报 |

## 4. Backend AI 架构

Backend 负责所有复杂的 AI 逻辑：

### 4.1 Model Router

根据任务类型、成本、可用性选择模型：

| 任务 | 首选模型 | 备选模型 | 原因 |
|------|---------|---------|------|
| Today One Thing 生成 | Claude / GPT-4 | GPT-3.5 / DeepSeek | 需要高推理能力 |
| Why Now 解释 | Claude / GPT-4 | GPT-3.5 | 需要可解释性 |
| AI Chat | Claude / GPT-4 | GPT-3.5 | 需要长上下文理解 |
| Memory 分析 | GPT-4o-mini / Claude Haiku | 本地小模型 | 高频、成本敏感 |
| 图片理解 | GPT-4o / Gemini | Claude | Vision 能力 |
| Weekly Brief | Claude / GPT-4 | - | 低频次，可用强模型 |

### 4.2 Prompt Management

- 所有 Prompt 集中在 Backend 管理。
- 按任务类型分层：Memory Analysis Prompt、Why Now Prompt、Chat Prompt、Weekly Brief Prompt。
- Prompt 版本化，支持 A/B Test。
- 支持按语言切换 Prompt（zh / en）。

### 4.3 Response Schema

Backend 返回结构化 JSON，Flutter 不解析自由文本。

Example Today One Thing Response：

```json
{
  "has_recommendation": true,
  "person_id": "uuid",
  "person_name": "Alice",
  "title": "45 天没有联系",
  "context": "她最近提到工作压力很大",
  "suggested_action": "发一句轻松的关心：最近怎么样？",
  "why_now": "你们过去平均每月联系 2-3 次，最近 45 天无互动",
  "priority": "high",
  "confidence": 0.92
}
```

### 4.4 Caching Strategy

- Today One Thing 每天计算一次，缓存 6-12 小时。
- Relationship Summary 每周更新一次。
- AI Chat 不缓存完整对话，但缓存上下文摘要。
- Memory Analysis 结果缓存，避免重复调用。

### 4.5 Fallback Strategy

| 失败场景 | Fallback |
|----------|----------|
| 模型不可用 | 切换到备选模型 |
| 所有模型不可用 | 返回本地规则生成的 Reminder（如生日提醒） |
| 响应解析失败 | 返回空，不展示错误给用户 |
| 超时 | 返回缓存结果或默认空状态 |

## 5. 模型选择策略

### 5.1 当前推荐模型

- **Primary：** Claude 3.5 Sonnet / GPT-4o
  - 擅长长上下文、关系理解、可解释性输出。
- **Cost-Optimized：** GPT-4o-mini / Claude Haiku
  - 用于高频、低复杂度的 Memory Analysis。
- **Vision：** GPT-4o / Gemini 1.5 Pro
  - 用于截图、图片理解。

### 5.2 预留模型

- **DeepSeek：** 国产模型备选，成本敏感时使用。
- **Qwen：** 中文场景备选，长文本处理能力强。

### 5.3 切换原则

- 模型切换不影响 Flutter。
- 切换基于：成本、效果、可用性、隐私合规。
- 保留模型 evaluation pipeline，定期对比不同模型在真实任务上的表现。

## 6. 隐私与数据安全

### 6.1 数据流向

```
Flutter → Backend → Prompt Engineering + 脱敏 → LLM
```

- Flutter 不直接向 OpenAI/Claude 发送数据。
- Backend 负责对敏感信息（真实姓名、地址、具体公司）进行脱敏或标记化处理。
- 用户可选择 Privacy Mode：敏感 Memory 不上云，仅本地处理。

### 6.2 数据保留

- LLM 调用日志不保留用户原始 Memory 明文。
- AI 输出结果保留在 Backend，用于缓存与 A/B Test。
- 用户删除数据后，所有相关 AI 上下文同步清除。

## 7. AI 能力边界

Flutter AI Layer 不实现以下能力，全部由 Backend 提供：

- 人格分析（MBTI/Big Five）
- 关系评分
- 冲突预测
- 自动发送消息
- 读取未授权聊天记录
- 情感诊断

LifeOS 的 AI 只提供：

- Today One Thing 建议
- Why Now 解释
- 基于授权 Memory 的 Chat
- 记忆分析后的 Reminder 生成
- 长期关系摘要

## 8. 与 PWA 的 AI 一致性

| 层级 | PWA | Flutter |
|------|-----|---------|
| Prompt 工程 | ✅ Backend 负责 | ✅ 复用 |
| 模型选择 | ✅ Backend 负责 | ✅ 复用 |
| AI Response Schema | ✅ Backend 定义 | ✅ 复用 |
| 业务逻辑 | ✅ Backend 负责 | ✅ 复用 |
| UI 展示 | React Components | Flutter Widgets |
| 本地缓存 | IndexedDB / localStorage | SQLite / Drift |

## 9. 总结

Flutter 的 AI 架构非常简单：**所有智能在 Backend，Flutter 只负责展示和交互。** 这种设计保证了：

- 模型可以自由切换。
- 成本可以统一管理。
- 隐私可以统一控制。
- PWA 和 Flutter 的 AI 体验完全一致。
- Flutter 启动时，PWA 已经验证过的 AI 能力可以直接复用。

> **Flutter 是 AI 的载体，不是 AI 的处理器。**
